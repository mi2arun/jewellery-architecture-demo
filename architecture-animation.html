POS / Invoice Inventory Posting API (double-entry) Physical Ledger (official books) Logical Ledger (complete truth) Personal (Logical only)
How to read this: Every non-personal business transaction is written directly & synchronously to the Physical Ledger, then an event fans it out to the Logical Ledger. Every transaction — business and personal — reaches the Logical Ledger via the event bus. A personal invoice skips Physical entirely and lands only in Logical. Therefore: Logical = Physical + Personal. Across all 50 stores, each ledger DB lives in its own geo-region.

Scene ⑤ — Secured Reporting & Mobile DB Switch: Reporting is split into two planes. Normal employees use the Physical Reporting API for everyday MIS & balance reports (read-only user physical_ro); they have no route to Logical. The Logical Ledger is reached only by management through a secured mobile app + Management Portal on a private network, behind step-up MFA, with every read written to a hash-chained audit log and personal rows masked unless authorized. Management uses a Management Web Portal (the reporting UI) — Top Management user → Web Portal → step-up MFA → Connection Router, with every read written to a hash-chained audit log and personal rows masked unless authorized. A separate routing-only mobile app connects directly to the Router; its single “Switch Connection” button toggles which DB the Router binds to — Physical (Region A, physical_ro) ↔ Logical (Region B, logical_ro) — one active connection at a time. The mobile app does routing only; it is not the reporting UI and never touches the employee plane.
